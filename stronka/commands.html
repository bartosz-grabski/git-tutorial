Git jest git!

Przyjdź i poznaj jeden z najpopularniejszych systemów kontroli wersji w praktyce.

Polecenia użyte podczas warsztatów

Część mniej zaawansowana

  • git init inicjalizacja repozytorium w bierzącym katalogu
  • git status aktualny stan katalogu roboczego (zmienione, dodane, usunięte pliki)
  • git add converter.py dodanie pliku converter.py do najbliższego commita (stage'owanie pliku)
  • git commit -m "commit message" commitowanie zmian
  • git remote add origin git@github.com:Mequrel/bit-workshop.git dodanie adresu repozytorium zewnętrznego
  • git clone git@github.com:Mequrel/bit-workshop.git skopiowanie repozytorium zewnętrznego
  • git pull [origin master] pobranie zmian z zewnętrznego repozytorium ( = git fetch + git merge)
  • git push [origin master] wypchnięcie naszych zmian do zewnętrznego repozytorium
  • git diff sprawdzenie różnicy pomiędzy ostatnim commitem a nie zastage'owanymi plikami
  • git log wyświetlenie historii commitów (tekstowo)
  • gitk wyświetlenie historii commitów (graficznie)
  • git mergetool uruchomienie narzędzia do rozwiązywania konfliktów
  • git commit --amend -m "Nowy commit message" dodanie zmian do poprzedniego commita i/lub poprawienie commit message

Część bardziej zaawansowana

  • git branch wylistowanie branchy
  • git checkout file-font-loader zmiana brancha na file-font-loader
  • git merge file-font-loader merge'owanie brancha file-font-loader do bieżącego brancha
  • git stash stashowanie zmian (odłożenie niescommitowanych zmian na stos)
  • git stash list wylistowanie stashów (snapshotów)
  • git stash apply zaaplikowanie stasha znajdującego się na wierzchu stosu
  • git stash drop usunięcie stasha ze stosu
  • git checkout -- font.py anulowanie wszystkich zmian w pliku font.py od czasu ostatniego commita
  • git reset HEAD~1 --soft anulowanie samego działania polecenia commit (zmiany w plikach zostają zachowane)
  • git reset HEAD~1 --hard anulowanie ostatniego commita (wraz ze zmianami które wprowadził)
  • git revert HEAD wygenerowanie commita, który odkręca wszystkie zmiany wprowadzone przez ostatni commit
  • git blame converter.py wyświetlenie historii pliku converter.py w taki sposób, by widzieć kto ostatni modyfikował daną linijkę
