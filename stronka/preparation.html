Git jest git!

Przyjdź i poznaj jeden z najpopularniejszych systemów kontroli wersji w praktyce.

Czego potrzebuje?

Obowiązkowo:
  • Laptop z dostępem do Internetu lub ewentualnie kolegę/koleżankę z laptopem jeśli zgodzi się pracować razem
  • Konto w serwisie GitHub
  • Zainstalowany Git
    • W przypadku Windowsa polecam GitHub for Windows. Jeśli macie już coś innego zainstalowanego to proszę o skonfigurowanie dostępu SSH do GitHuba. Niezbędny jest też dostęp do konsoli gitowej.
    • W przypadku linuksa sprawa powinna być prostsza i sprowadza się zazwyczaj do uruchomienia apt-get install git albo yum install git-core.
  • Zainstalowany Gitk
    • W przypadku Windowsa instalowany jest od razu z Gitem.
    • W przypadku linuksa ściągamy go np. za pomocą apt-get install gitk
  • Skonfigurowanie dostępu przez SSH do GitHuba (instrukcja) lub ominięcie podawania hasła za każdym razem w dowolny inny sposób. (W przypadku GitHub for Windows wystarczy tylko raz się zalogować do programu)
  • Zainstalowane narzędzie do rozwiązywania konfliktów. Dla Linuksa polecam melda (w przypadku debianów wystarczy apt-get meld). Dla Windowsa z kolei Kdiff3
  • Ulubiony edytor tekstu (jeśli nie masz jeszcze ulubionego to polecam Notepad++ albo Sublime Text)
Wskazane lecz opcjonalne:
  • Zainstalowany interpreter języka python w wersji 2.7, pobrany np. tutaj, albo zainstalowany przez manager pakietów (chocolatey, apt-get, itd.) - koniecznie sprawdzić czy działa! (uwaga: znajomość tego języka jest kompletnie nieistotna)
