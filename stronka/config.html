Git jest git!

Przyjdź i poznaj jeden z najpopularniejszych systemów kontroli wersji w praktyce.

Konfiguracja

Konsola

  1. Uruchamiamy konsolę gitową (Git Shell w GitHub for Windows) lub zwykłą konsolę na linuksie

Git

  1. Konfigurujemy nasze dane, którymi podpisywać będziemy commity: 
    ~ git config --global user.name "Mequrel"
~ git config --global user.email "me@here.com"
  2. Konfigurujemy narzędzie do rozwiązywania konfliktów (przykład dla kdiff3): 
    ~ git config --global merge.tool kdiff3
~ git config --global mergetool.kdiff3.path 'C:\Program Files (x86)\KDiff3\kdiff3.exe'

Repozytorium na GitHubie

  1. Jedna osoba z pary tworzy nowe repozytorium.
  2. Dodaje też drugą osobę jako współpracownika i daje mu prawa do edycji

Skrypt `mayday`

  1. Pobieramy paczkę ze skryptem:
    • Wersja dla systemu Windows
    • Wersja dla systemu Linux
  2. Edytujemy plik config.ps1 (Windows) lub config.sh (Linux) i podajemy adres SSH stworzonego repozytorium: 
    $github_url = "git@github.com:Mequrel/bit-workshop.git"    [WINDOWS]
github_url="git@github.com:Mequrel/bit-workshop.git"       [LINUX]
  3. Użycie: 
    ~ .\mayday.ps1 N U    [WINDOWS]
~ ./mayday.sh N U     [LINUX]
    Gdzie N oznacza nr stanu, do którego chcemy przywrócić nasze repozytorium, a U (0 lub 1) oznacza czy mamy aktualizować zewnętrzne repozytorium.
    UWAGA! Jeżeli pracujesz w parze, to druga osoba musi również uruchomić skrypt z tym samym parametrem. Jedna osoba z pary powinna mieć U=1

Stworzenie katalogów roboczych

  1. Uruchamiamy: 
    ~ .\mayday.ps1 0 0    [WINDOWS]
~ ./mayday.sh 0 0     [LINUX]
  2. Przechodzimy do katalogu ./gitlabs/1 lub ./gitlabs/2 w zależności od naszej roli w parze.
